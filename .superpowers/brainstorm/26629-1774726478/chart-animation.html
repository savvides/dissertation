How should the charts update as the user scrolls?

For the scrolltelling narrative, how interactive do you want the visual elements to be?

A

Sticky Chart & Replacing Data

A single chart area stays pinned to the right (or top on mobile) while the text scrolls on the left. As the user reaches new sections, the chart smoothly morphs to show new data (e.g., from learning gains to presence to cognitive load).

B

Inline Reveal & Animation

Charts are embedded directly in the text flow. As the user scrolls down, the charts animate into view (bars growing, data points fading in) at the exact moment they are mentioned in the text.

C

User-Triggered Exploration

Charts are embedded in the text, but wait for user interaction to animate or reveal findings. The user clicks a button like "Reveal Result" to trigger the animation, making them an active participant in discovering the findings.
